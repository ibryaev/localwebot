### Функционал

Этот бот позволяет объединить несколько чатов в одну паутину, позволяя централизировать несколько проектов и объединяться с другими чатами.

> [!NOTE]
> Бот также может быть и просто модератором, не обязательно иметь сетку чатов.

* Глобальная администрация и её иерархия:
  * 1 **Модер** может глобально мутить и глобально кикать, решать жалобы и удалять сообщения (встроенной командой).
  * 2 **Админ** наследует права модера и может их наказывать, может глобально банить.
  * 3 **Хелпер** - по сути мини-владелец. Наследует права модера и админа, но не может быть кем-либо наказан (включая владельцем).
    Может добавлять новых модераторов и повышать их до админов.  
    Владелец чата, чей чат есть в сетке, не может быть в ней понижен.
  * **Наследник паутины** унаследует права владельца на паутину, если тот пропадёт (удалиться ТГ-аккаунт).
    Это не отдельная должность - наследником может быть хоть модератор, его могут наказывать и разбирать на него жалобы, но снять с него админские права может только владелец.
  * 4 **Владелец паутины** имеет все права, включая удаление самой паутины, назначение и снятие её наследника, привязка админского чата для всей сетки.
* Глобальные наказания:
  * **Глобальный бан** мгновенно банит пользователя во всех чатах этой сетки, в которых он состоял. А при заходе в другой чат, то бот его забанит и там.
  * **Глобальный мут** мгновенно запрещает все права этому пользователю (включая создание опросов)
    во всей чатах этой сетки, в которых он состоял. А при заходе в другой чат, то бот его забанит и там.
  * **Глобальный кик** мгноваенно кикает пользователя из всех чатов этой сетки, в которых он состоя.
* **Глобальные жалобы**.
  В любом чате сетки можно написать команду "жалоба" (и, опционально, дать причину) и в админский чат этой паутины придёт уведомление с краткой сводной и ссылкой на сообщение жалобы.

> [!IMPORTANT]
> Один человек может иметь одну паутину.  
> Один чат может состоять в одной паутине.

### Сборка

1. Минимальная требуемая версия Python - 3.12. Рекомендуемая - 3.13 или 3.14.
2. Клонируйте репозиторий:
    ```Terminal
    git clone https://github.com/ibryaev/localwebot
    cd localwebot
    ```
3. Создайте свою posgtre базу данных из **скрипта** [schema.sql](src/database/schema.sql) и запустите сервер **перед** запуском бота.
4. Установите зависимости в вирутальное окружение:
    ```Terminal
    python3 -m venv .venv
    source .venv/bin/activate
    pip install -r requirements.txt
    ```
5. Запустите проект через файл [main.py](src/main.py). В случае успеха в терминале выведется "1".
